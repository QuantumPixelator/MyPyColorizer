# Contributing to MyPyColorizer

Thank you for your interest in contributing to MyPyColorizer! This lightweight color library welcomes contributions from developers of all skill levels.

## Code of Conduct

Be respectful, constructive, and patient. This project aims to be beginner-friendly.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/yourusername/mypycolorizer.git
   cd mypycolorizer
   ```
3. **No additional setup required** - this is a pure Python package

## Development Guidelines

### Code Style
- Follow PEP 8
- Keep the library lightweight
- Maintain backward compatibility
- Add clear docstrings

### Adding Colors/Styles
- Use standard ANSI escape codes
- Test across different terminals
- Document the new addition
- Update examples if needed

### Commit Messages
- Clear and descriptive
- Start with action verb
- Example: `Add: Support for new color styles`

### Pull Requests
- Reference issue numbers
- Include testing details
- Describe terminal compatibility
- Provide usage examples

## Testing Your Changes

1. Test in multiple terminals:
   - Windows Command Prompt
   - Windows PowerShell
   - Windows Terminal
   - Linux/macOS terminals

2. Verify ANSI codes work correctly

3. Test all existing functionality still works

## Beginner-Friendly Tasks

- Add new color constants
- Improve documentation
- Add more usage examples
- Create test scripts
- Research terminal compatibility

## Terminal Compatibility Notes

Different terminals support different ANSI codes:
- Windows Terminal: Full support
- Windows PowerShell: Most support (may need enabling)
- Command Prompt: Limited support
- Linux terminals: Usually full support

Always test your changes across platforms!

## Need Help?

- Check existing issues
- Ask questions in PR comments
- Look at the current code structure
- Start with small, focused changes
